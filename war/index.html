Mnd.ly URL Redirector

This app makes some useful urls and resources out of research article identifiers and Bitly hashes. The aim was to make better persistent urls to point to our article pages than the current title slugs we generate by using existing identifiers for research articles that are used by other publications and organizations. The resulting app can redirect or generate resources for a range of identifiers that the Mendeley OAPI can support.

To be able to have a short url, forwarding requests to bit.ly was added so that we can replace the mnd.ly short url service with a combination of the two. As we can't move that over without more work this has been setup using the r.mensearch.net domain for the demo.

Usage

This app supports the following url schemas :-

  • Bitly - http://r.mensearch.com/{bitly-hash}
  • Article id - http://r.mensearch.com/{type}/{identifier}
    •

    Where {type} can be any identifier supported by Mendeley, currently: arxiv, doi, isbn, pmid, scopus or ssm

    Outputting the following resources :-

  • redirect
  • json
  • qr
    •

    These are used by adding ?format={format} to the url, with redirect being the default when nothing is specified.

    Some examples

    http://r.mensearch.net/pmid/19782018
    http://r.mensearch.net/danharvey?format=json
    http://r.mensearch.net/pmid/18492790?format=qr

    or some QR images :-

    pmid: 18492790 arXiv: 1104.2948 doi: 10.1016/j.cub.2011.03.017
